Async Processes & Communication

Both API and Worker are single-threaded asyncio processes — concurrency via coroutines, not threads.

API (FastAPI)

  • HTTP handlers — create, stop, resume, list
  • _profiling_listener — awaits jobs.profiling_complete, stores result, re-schedules
  • _completed_listener — awaits jobs.completed, schedules waiting jobs
  • _queue_watcher — 60 s poll safety net

schedule_lock (asyncio.Lock) serialises all scheduling decisions across coroutines.

Worker

  • _handle_job_submitted — awaits jobs.submitted, enqueues job
  • _handle_stop_requested — awaits jobs.stop_requested, sends SIGTERM
  • _handle_resume_requested — awaits jobs.resume
  • _job_runner — dequeues and runs Docker containers concurrently
  • _run_job — one task per container: exec, stream logs, report completion

Inter-process flow

HTTP request  →  API publishes NATS event  →  Worker picks up
Worker finishes →  publishes completed/profiling_complete  →  API listener reacts
All scheduling goes through schedule_lock  →  reads GPU usage from DB  →  assigns node

Profiling Scheduler

Goal: find the fastest GPU configuration per job through incremental measurement.

Exploration

  • Enumerate configs as cartesian product of profiling-node GPU counts
  • Intersect with configs production nodes can run
  • Sort by total GPU count ascending
  • Pick next un-profiled config → run on profiling node
  • Record (gpu_config, duration_seconds)

Exploitation

  • All configs profiled → pick first with available production node
  • Duration recorded but not yet used for config ranking
  • Warmup-aware estimation: drop first inter-step interval, average the rest

Profiling execution

MAX_STEPS = profiling_epochs_no   (default 100)
LOG_INTERVAL = per-job setting    (default 50 → 2 intervals, configurable lower)
Checkpoint dir: data/checkpoints/{id}/.profiling/  (isolated, cleared each run)

Worker reports result via jobs.profiling_complete → API re-queues job. When a job completes (jobs.completed), the API immediately tries to schedule waiting jobs; 60 s watcher as safety net.
